Unicode reference

Unicode is the most primitive application of fonts on the web, with the following features:

  • Best compatibility, supports IE6+, and all modern browsers.
  • Supports dynamic adjustment of icon size, color, etc. by font.
  • But because it's a font, it doesn't support multicolor. Only single-color icons in the platform can be used, even if there are multi-color icons in the project, they will be automatically decolorized.

Note: The new version of fonticon supports multi-color icons. These multi-color icons cannot be used in Unicode mode. If there is a need, it is recommended to use the symbol reference method

The steps for using Unicode are as follows:

Step 1: Copy the @font-face

generated under the project 
@font-face {
  font-family: 'fonticon';
  src: url('fonticon.eot');
  src: url('fonticon.eot?#iefix') format('embedded-opentype'),
      url('fonticon.woff2') format('woff2'),
      url('fonticon.woff') format('woff'),
      url('fonticon.ttf') format('truetype'),
      url('fonticon.svg#fonticon') format('svg');
}

Step 2: Define styles that use fonticon

.fonticon {
  font-family: "fonticon" !important;
  font-size: 16px;
  font-style: normal;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

Step 3: Select the corresponding icon and get the font code to apply to the page

<span class="fonticon">&#x33;</span>

"fonticon" is the font-family under your project. It can be viewed by editing the project, the default is "fonticon".

font-class reference

font-class is a variant of Unicode usage, mainly to solve the problem of unintuitive Unicode writing and unclear semantic meaning.

Compared with Unicode usage, it has the following characteristics:

  • Good compatibility, supports IE8+, and all modern browsers.
  • It is more intuitive to write than Unicode semantics. It's easy to tell what this icon is.
  • Because classes are used to define icons, when you want to replace icons, you only need to modify the Unicode references in the class.
  • However, multi-color icons are not supported because they are essentially the same fonts used.

The steps are as follows:

Step 1: Import the fontclass code generated under the project:

<link rel="stylesheet" href="./fonticon.css">

Step 2: Select the corresponding icon and get the class name to apply to the page:

<span class="fonticon icon-xxx"></span>

" fonticon" is the font-family under your project. You can view it by editing the project, the default is "fonticon".

Symbol reference

This is a brand-new way of use. It should be said that this is the mainstream in the future, and it is also the currently recommended usage of the platform. For more information, please refer to this article This usage is actually a collection of SVG, which has the following characteristics compared with the other two:

  • Support multi-color icons, no longer limited by single color.
  • With some tricks, it is supported to adjust styles like fonts through font-size, color.
  • Poor compatibility, supports IE9+, and modern browsers.
  • The performance of browser rendering SVG is average, not as good as png.

The steps are as follows:

Step 1: Import the symbol code generated under the project:

<script src="./fonticon.js"></script>

Step 2: Add common CSS code (introduce once):

<style>
.icon {
  width: 1em;
  height: 1em;
  vertical-align: -0.15em;
  fill: currentColor;
  overflow: hidden;
}
</style>

Step 3: Select the corresponding icon and get the class name to apply to the page:

<svg class="icon" aria-hidden="true">
  <use xlink:href="#icon-xxx"></use>
</svg>
